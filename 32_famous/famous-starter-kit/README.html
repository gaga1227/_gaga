Famo.us Starter Kit

Famo.us is a JavaScript platform that helps you build beautiful, cross-platform web apps.

It is the only framework that provides an open source 3D layout engine fully integrated with a 3D physics-based animation engine that can render to DOM, Canvas, or WebGL.

Getting started is easy. In addition to this starter kit, these resources are available:

  • Famo.us University: Learn to code in Famo.us with lessons, live code examples, and real app projects on http://famo.us/university.

  • IRC channel: Visit #famous on irc.freenode.net for live help any time.

  • Bug reports: Send bug reports to bugs@famo.us.

  • GitHub: Visit http://github.com/famous/famous to show your support with a star.

  • Twitter: Follow us at http://twitter.com/befamous

  • Facebook: Like our page at http://facebook.com/famousindustries

Contents

boilerplate

The boilerplate is a tiny snippet which shows you the basic structure of an application.

Open it in any browser window by double-clicking or navigating to boilerplate/index.html.

interface-demos

Members of our community have also tried their hand at building impressive interactions like those seen on many of your favorite mobile apps. The interface-demos folder has completed interactions demonstrating many of the interactions you can create in Famo.us. Open one up, like interface-demos/Airbnb/index.html, right in your browser, to see what that would look like on a mobile device. Or, use your own mobile device to directly open the neighboring app.html.

reference-docs

Building an application like those in the tutorials require you to understand the details of many smaller components. Some of these have their own lessons in Famo.us University, but you can always look here to find the details of any class or method.

Start browsing by opening reference-docs/(some version)/index.html.

reference-examples

The documentation on the methods in Scrollview is useful, but often you may choose to get started by just putting one on the screen and playing. The reference-examples folder shows small examples for most of our classes in the framework. Start playing by opening any .html file, such as reference-examples/views/Scrollview/example/index.html. The neighboring src directory implements the example you see.

These examples are also available on http://famo.us with links from their respective documentation files, or directly in a list at http://famo.us/examples.

reference-faq

In the reference-faq folder, you'll find a high level document that helps answer questions like:

  • What is Famo.us all about?
  • What can I build with Famo.us?
  • What's the best way to learn Famo.us?
  • What browsers work with Famo.us?
  • What devices and platforms does Famous work on?
  • What third-party libraries work with Famo.us?

You don't have to read it to get started, but this will help answer some higher-level questions.

reference-guides

For larger technical topics not covered in the very detailed documents, consult some of our guides at guides/index.html. These cover topics like:

  • the Render Tree
  • using our eventing system, and
  • common pitfalls.

reference-tutorials

Famo.us University (http://famo.us/university) offers some more in-depth tutorials that require you work on your own computer rather than in the Famo.us University code editor.

The reference-tutorials folder contains starter kits for each of the tutorials on Famo.us University. Going through a tutorial like http://famo.us/university/famous-101/slideshow requires these assets on your computer, like the beginning structure of the app, images used by the app, and so on.

Opening a folder like reference-tutorials/slideshow will provide a brief README.html referencing the Famo.us University content it goes with.

In that folder, you should check out final/index.html to see where you will end up when you're done with any tutorial. Don't cheat and look at the code yet, though. It's basically an answer key, but you should start at the start (directory).

Inside that folder, the start folder has an index.html file which you can open, which will typically show a blank page. No worry. You will start build Famo.us code on top of this in the start/src directory to end up with the result in final.
