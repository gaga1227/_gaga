IE hasLayout Explained

Last update: 2011-07-04

Problem:

hasLayout issue with ordered list in IE6,7

Symptoms:

  • Only happens with ordered list in IE6, 7
  • The list item number always shows 1 for all items, not counting
  • The list item number is showing at the bottom of the li elements instead of top

Cause:

The ordered list item has “hasLayout” turned on.
It can be turned on by:

  • position: absolute
  • float: left | right
  • display: inline-block
  • height: any value other than 'auto'
  • zoom: any value other than 'normal' (MS proprietary)
  • writing-mode: tb-rl (MS proprietary)

As of IE7, overflow became a trigger for hasLayout.

  • overflow: hidden | scroll | auto

Fix:

Make sure "hasLayout" is off for ordered list items
